Project Proposal: LLM Document Reconstruction
Automated SEC Disclosure Reconstruction via LLM Agents
1. Objective
To evaluate the efficacy of Large Language Models (LLMs) in reconstructing
structured financial data from semi-structured raw SEC filings (EDGAR), utilizing
HyperStreamDB—the researcher’s open-source Rust-based database engine—as
a high-performance, transactional storage and retrieval engine. GitHubLinks to an external site. | PyPILinks to an external site. |

2. Research Hypothesis
“An LLM-driven inference agent, when provided with a deterministic Markdown
representation of a raw SEC filing (formatted via edgartools or other similar
library), can extract and reconstruct the original transaction records with >95%
accuracy compared to a ground-truth CSV, while capturing nuanced semantic
context that traditional hard-coded parsers miss.”
3. Methodology
Phase 1: Raw Data Acquisition & Normalization
•Source: Fetch original .xml and .html files (Form 4) from the SEC EDGAR
archive. Candidates for extraction include sec-edgar-downloader,
edgartools, and the author’s own openedgarLinks to an external site. library, depending on the
efficiency of the raw byte-stream handling.
•Normalization Strategy: Convert raw filings into Structured Markdown
using an industry-standard library. Candidates for evaluation include:
–edgartools.markdown() (Deterministic financial parser).
–Microsoft MarkItDown (Multi-format conversion engine).
–Unstructured.io (Semantic layout-aware partitioning).
–sec-parser (Specialized SEC-specific parsing).
–LlamaParse / Markdownify (General PDF/HTML to MD conversion).–
Rationale: Deterministic parsing ensures that the LLM receives the
maximum possible structural context (Headers, Tables, and
Footnotes) without hallucinating the original document’s layout.
Phase 2: Agentic Extraction (The “AI” Layer)
•
Process: Feed the Markdown-normalized context into a cost-effective
Open Source LLM. High-performance candidates for local or hosted
inference include:
–Llama 3.1 / 3.3 (High reasoning for transaction logic).
–DeepSeek-V3 / DeepSeek-R1 (Highly efficient for structured
extraction).
–Qwen 3.5 / 3.6-Plus (The latest SOTA for multi-modal and agentic
extraction as of April 2026).
–Note: Utilizing open-source weights (via Ollama or Groq) ensures
project scalability and budget control compared to the high token
costs of proprietary APIs (GPT-4o/Claude).
•Task: The LLM acts as an “Extraction Agent” with a strict JSON schema
prompt to reconstruct the transaction data.
•Target Fields: Issuer Ticker, Owner Name, Transaction Code, Shares,
Price, and Full Footnote Text.
Phase 3: High-Performance Storage (The “Systems” Layer)
•Structured Ingestion: Ingest the LLM-extracted JSON records into
HyperStreamDB (the author’s custom-built high-performance engine).
•Ground Truth Persistence: Store each normalized SEC filing as its own
persistent Markdown record.
•Cross-Referencing: Include a column in the database record (keyed by
Accession Number) that references the corresponding Markdown file path,
allowing for “In-Index” AGENT reasoning over the source document.
•Architectural Design Question: Evaluate the storage trade-offs between
storing Markdown as a Binary Blob (Packed Bitmap) for space efficiency
versus a Raw Text File for direct Large Context Window accessibility.
•Indexing: Index the extracted LLM context using HNSW vector search
and the transaction metadata using RoaringBitmap scalar filtering.
Phase 4: Evaluation & Benchmarking
•
Field Accuracy: % of identical matches for deterministic fields (e.g.,
Transaction Price, Quantity).•Semantic Fidelity: Use BERTScore to compare the LLM’s extracted
footnote text against the CSV.
•System Performance: Measure the Ingestion Latency and Hybrid
Retrieval (Vector + Scalar) speed of HyperStreamDB on the SEC dataset.
4. Required Tech Stack
•DB Engine: HyperStreamDB (Rust-based with Python wrapper).
•Infrastructure Context: HyperStreamDB uses Arrow-native ingestion
and Iceberg-style manifests for ACID transactional consistency on
document-based data lakes.
•Language: Python 3.10+.
•Libraries: BeautifulSoup4, sec-edgar-downloader, pandas, maturin (for
HyperStreamDB bindings).
•APIs: OpenAI/Anthropic for the extraction agent.
